Automation Scope

WhatsApp Outreach Automation

Excel-driven outreach through an active WhatsApp Web session.

A desktop automation system that processes customer rows from Excel, sends one controlled WhatsApp message at a configured interval, and writes the result back to the same operational report.

No API Uses a normal WhatsApp number already logged in on the laptop. No Meta Business API or WhatsApp API dependency.
5 min Default spacing between messages to reduce risk and keep sending behavior human-paced.
Excel Rows remain the source of truth for status, attempts, sent message, notes, and customer IDs.

Operating Flow

01Open Excel and select the next row marked Pending or Retry.
02Validate and normalize the phone number, then create a missing customer ID.
03Open the WhatsApp chat, confirm availability, and prepare the message.
04Send, update the row status, wait, then move to the next eligible row.

Workflow + Controls

Message System

Two sending modes for different campaign needs.

AI Variation Mode

  • Keeps the same business intent while changing wording per recipient.
  • Can include name, unit, property, or location when available.
  • Uses professional language with no fake promises, invented details, or aggressive copy.

Fixed Message Mode

  • Admin writes the exact message once.
  • The same approved wording is sent to every selected customer.
  • Best for controlled announcements, follow-ups, and approved scripts.

Admin Controls

Operational control without manual sending.

  • Select the Excel file and start, pause, resume, or stop automation.
  • Edit message context, choose message mode, and set delay between messages.
  • View pending, sent, failed, invalid, and skipped counts.
  • Retry failed rows, skip selected rows, and export the updated Excel report.

The automation should use browser automation against WhatsApp Web selectors with explicit state tracking. Blind mouse-click automation is avoided because it is fragile and difficult to recover when the UI changes.

Setup + Safety

Laptop Environment

Runs continuously on a dedicated Windows laptop.

  • WhatsApp Web or Desktop stays logged in with the active sending number.
  • Stable internet, accessible Excel file, and the automation app remain running.
  • Laptop sleep is disabled or handled so the process can run 24/7.
  • If the laptop restarts, processing resumes without duplicate sends.

Contact Handling

Phone validation, availability checks, and duplicate prevention.

  • Normalize phone numbers before attempting WhatsApp chat creation.
  • Generate missing internal IDs and preserve existing IDs.
  • Mark numbers as Invalid Number or Not on WhatsApp when required.
  • Do not send again to rows already marked Sent unless an admin manually retries.

Error Handling

Recoverable states are logged back to Excel.

  • WhatsApp logged out, internet disconnected, WhatsApp loading delays, or send failures.
  • Invalid phone numbers, missing WhatsApp accounts, duplicate rows, and already-sent customers.
  • Excel file locked, unavailable, or temporarily blocked by another application.
  • Each failure stores status, timestamp, and notes for review or retry.

Risk Controls

Safety rules reduce account-ban exposure.

  • Send at a slow interval, defaulting to one message every five minutes.
  • Avoid spammy wording, links in the first message, and high daily volume.
  • Use warm numbers, human wording, opt-out tracking, and stop on negative replies.
  • Do not message random scraped numbers or contacts without a clear business reason.

WhatsApp cannot force-register a number that is not on WhatsApp. The system can only mark the number unavailable or optionally save a local contact record. If industrial-scale outreach is required, the correct tool is the official WhatsApp Business API, not a normal WhatsApp number.
