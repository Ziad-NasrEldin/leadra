[Heading1] 1. Product Overview
Leadra needs an internal real estate resale management system for sales representatives, managers, sub-admins, and admins. The system helps Leadra manage resale units across multiple real estate projects, developers, destinations, and locations in Egypt.
The first delivery phase is a mobile-first website. Later, the system can be wrapped using Capacitor or another approved wrapper for mobile app publishing, including iPhone sales app distribution if approved.
The first phase must prioritize mobile usage because most daily work is expected to happen from phones. Desktop must remain usable, but advanced desktop optimization is not the priority for the first delivery.
[Heading1] 2. Product Goal
Build a simple internal system where Leadra can:
[ListBullet] Add resale units.
[ListBullet] Manage unit details.
[ListBullet] Protect owner and sales-sensitive information.
[ListBullet] Track sales team activity.
[ListBullet] Manage users, roles, teams, branches, and permissions.
[ListBullet] Manage Developers, Destinations, Projects, Views, Teams, Branches, and PDF configuration.
[ListBullet] Export branded permission-safe unit PDFs.
[ListBullet] Share generated PDFs through available device or browser sharing options.
[ListBullet] Track all important actions in an audit log.
[ListBullet] Notify users inside the system and by email.
[ListBullet] Reduce duplicate owner records using normalized phone numbers.
[ListBullet] Support project/location-first unit browsing with separate pages after selection.
[ListBullet] Support a Payments Tab with calculated Remaining Value and payment timetable control.
[ListBullet] Preserve unit ownership history when sales representative accounts are deleted or reassigned.
[ListBullet] Allow multiple active sessions across devices unless revoked by an approved security or admin action.

[ListBullet] Installment Amount = Remaining Value divided by number of scheduled payments.
[ListBullet] Quarterly = 4 payments per year.

[Heading1] 32. Core User Flows
Create Unit Flow
1. User logs in.
2. User clicks Add Unit.
3. User fills unit form.
4. User selects Destination, Developer, Project, Unit Type, and conditional area fields.
5. System shows only fields relevant to selected Unit Type.
6. User selects payment method: Cash or Installment.
7. User fills pricing fields based on selected payment method.
8. User sets installment period From/To if applicable.
9. User selects owner country code.
10. User enters owner phone.
11. System normalizes phone number.
12. System validates required fields and required-field stars.
13. System checks duplicate owner phone rules using normalized phone.
14. System blocks creation only if the same normalized phone exists inside the same project.
15. System allows same normalized phone in other projects silently.
16. System calculates initial Remaining Value if payment method is Installment.
17. System calculates installment amount if installment type is Quarterly, Semi-Annual, or Annual.
18. System generates unit code.
19. User uploads allowed images and permitted PDF attachments only; videos are rejected.
20. System validates media count, total media size per unit, related-photo requirement for PDFs, and that no videos are included.
21. User submits unit.
22. System creates unit.
23. System sends in-app and email notifications.
24. System records action in audit log.
View Unit Flow
1. User opens View All Units.
2. User sees project/location cards.
3. User selects project/location card.
4. System opens a separate page for that project/location.
5. User sees units inside selected project/location.
6. User applies filters if needed.
7. User selects unit.
8. System opens Unit Details.
9. System hides or shows sensitive data based on permission.
Export PDF Flow
1. User opens Unit Details.
2. User selects which images should appear in PDF if user has edit permission.
3. User clicks Generate PDF or Export PDF.
4. System checks user permission.
5. System hides restricted information if needed.
6. System excludes images marked Hide from PDF and validates that no video media exists because videos are not allowed.
7. System generates branded PDF.
8. User downloads PDF.
9. System records export action.
10. System sends Admin/Sub Admin notification.
Share Generated PDF Flow
1. User opens Unit Details.
2. User generates PDF.
3. User clicks Share PDF.
4. System opens native/device share options if supported.
5. User chooses WhatsApp, Instagram, or another app.
6. If sharing is unavailable, user can download PDF and send manually.
Archive Unit Flow
1. Authorized user opens Unit Details.
2. User clicks Archive Unit.
3. System asks for confirmation.
4. System archives unit.
5. Unit no longer appears in active lists.
6. Unit remains stored in database.
7. Action appears in audit log.
Sales Representative Deactivation/Reassignment Flow
1. Admin/Sub Admin opens User Management.
2. Admin/Sub Admin selects Sales Representative to deactivate/delete.
3. System checks whether the Sales Representative has assigned or uploaded units.
4. If units exist, system requires selecting a replacement Sales Representative.
5. System reassigns units to the replacement Sales Representative.
6. System preserves prior assignment history.
7. System completes deactivation/deletion.
8. System records action in audit log.
[Heading1] 33. Business Rules
Business rules are intentionally not yellow highlighted, per Leadra instruction.
1. Users cannot register themselves.
2. Any user can create units.
3. All roles can view all units.
4. Owner phone is visible only to Admin, Sub Admin, and the Sales Representative who uploaded the unit.
5. Owner phone search is available only to Admin and Sub Admin.
6. Sales users cannot archive units.
7. Managers cannot archive units.
8. Units are archived, not permanently deleted.
9. Sales users cannot edit owner name or phone after creation.
10. Sales users cannot see owner phone for units uploaded by other sales users.
11. Sales users cannot search owner phone.
12. Sales users can see who uploaded a unit globally.
13. Admin and Sub Admin have the same access level unless a future approved change separates them.
14. Admin/Sub Admin can create and manage Destination, Developer, and Project.
15. Destination must remain in the system and in PDF output.
16. Owner duplicate checking uses normalized owner phone only.
17. Owner name must not trigger duplicate blocking.
18. Same normalized owner phone cannot be duplicated inside the same project.
19. Same normalized owner phone in another project is allowed silently.
20. Unit code must be generated automatically.
21. Unit code format is Project Abbreviation + Number of Bedrooms + BR.
22. Unit Type must use the fixed 12-type list.
23. Unit Type controls which area and floor fields appear.
24. For floor-based unit types, Floor supports Ground and floors 1 through 40.
25. Ground selection shows Garden Area where applicable.
26. Penthouse wording must be used globally for top-level/penthouse labels.
27. Penthouse Unit Type shows BUA and Terrace Area only.
28. Cabin shows BUA only.
29. Each unit can have only one payment method: Cash or Installment.
30. Payment method is shown in Unit Details but excluded from generated PDF.
31. Cash price is the same as Total Value.
32. Remaining Value initial calculation is Total Unit Cost minus Down Payment.
33. Remaining Value must not be manually edited.
34. Remaining Value changes only through the payment timetable after creation.
35. Admin/Sub Admin and the Sales Representative who uploaded the unit may edit Total Value / pricing where applicable, but those edits must not change Remaining Value outside the payment timetable.
36. Installment period uses From/To month-year selectors instead of numeric Installment Years.
37. Installment amount must be calculated automatically for Quarterly, Semi-Annual, and Annual payment options.
38. Custom installments must not be automatically calculated unless future configuration allows it.
39. Custom installment text must appear in the PDF when selected.
40. Fixed commission defaults to 1.5% unless configured otherwise.
41. Fixed commission can only be changed by Admin/Sub Admin.
42. Transfer fees show only if applicable.
43. Hold status does not expire automatically.
44. Sold status must distinguish Sold by Us and Sold by Others.
45. First uploaded image becomes thumbnail.
46. Videos are not allowed in unit media, galleries, thumbnails, generated PDFs, or PDF references.
47. Users can choose whether selected images appear in PDF.
48. PDF export must be branded and include the required unit, area, pricing, delivery, and custom installment data.
49. PDF sharing must be available after PDF generation where browser/device supports sharing, but web sharing cannot guarantee direct sharing to every app; PDF download fallback is required.
50. Notifications must be in-app and email.
51. Inactive users must remain visible in audit history.
52. Every sensitive action must be tracked in audit log.
53. Media upload limit starts at 40 MB total per unit.
54. Upload failure must show a clear error and must not fail silently.
55. Delivery Date / Delivery Expectancy uses year-only selection unless Leadra later approves month-year delivery display again.
56. Delivery expectancy must support future years beyond next year.
57. Mobile UI has priority over desktop UI in first phase.
58. Major screen order and field order should match the PRD unless approved otherwise.
59. Users cannot edit their own passwords through self-service. Admin/Sub Admin can edit/reset user passwords through User Management or another approved administrative process.
60. The same user may keep multiple active sessions across devices unless revoked by admin or security policy.
61. Deleting or deactivating a Sales Representative must require reassignment and must never delete, archive, or hide assigned units.
62. Total Value / pricing after creation is editable only by Admin, Sub Admin, and the Sales Representative who uploaded the unit.
63. Destination must appear before Developer in Create Unit and Unit Details.
64. Furnished must be shown only when the unit is furnished; if not furnished, the field must not appear.
65. Finishing Status is required and must show a required-field star.
66. Unit Details must show Delivery before Commission.
67. Videos must be rejected at upload and must not be stored, shown, thumbnailed, exported, or referenced.
68. Admin/Sub Admin can reset/edit user passwords, while direct self-service password editing remains disabled.
